## 4. Known issues / open work

### 4.a Trained baselines return 0% — **FIXED in PR #13** (2026-05-02)

All three sub-bugs are resolved on the
`devin/1777721760-trained-baselines-prior-graph` branch and validated
on a live YandexGPT-Lite N=50 bench (412.52 ₽); see `docs/final_report.md`.

1. **Empty initial graph — fixed.** New `flybrain_prior_graph(agent_names)`
   in `flybrain/baselines/graphs.py` loads the on-disk K=64 FlyWire 783
   prior (`data/flybrain/fly_graph_64.fbg`, 64 nodes / 199 edges,
   modularity Q = 0.6800), aggregates the cluster-level adjacency into
   the 15-agent space by rank-of-degree round-robin, and emits a single
   fallback edge to the highest-degree hub for any agent that would
   otherwise be orphaned. `_flybrain_with_checkpoint` and
   `_flybrain_prior_untrained` now both ship this graph.
2. **Embedding-dimension mismatch — fixed.** Training scripts gained
   `--emb-dim`, `--graph-hidden-dim`, `--graph-out-dim`, `--fly-dim`,
   `--hidden-dim` flags with defaults pinned to the registry
   (32 / 16 / 32 / 8 / 32). `load_state_dict(strict=False)` no longer
   silently drops weights at registry-load time.
3. **Checkpoint env-var requirement — fixed.** `_flybrain_with_checkpoint`
   now auto-discovers `data/checkpoints/{label}_gnn.pt` when the
   matching env var is unset.

expanded fixtures**, 120 tasks per baseline overall — round-3

### 4.b Budget tracker overshoot at high parallelism

`BudgetTracker(hard_cap_rub=N)` is enforced per-LLM-call: a call that
*would* push cumulative spend over `N` raises `BudgetExceededError`.
But at `--parallelism 4`, in-flight tasks already started on workers
keep going. Last run requested 1000 ₽ cap, actually spent 1518 ₽
(~50 % overshoot).

**Fix:** add a `pre_emptive_cap_pct` param (default 0.7) so the
tracker halts at 70 % of the hard cap, leaving headroom for in-flight
workers. Or add a `--strict-budget` flag that runs at parallelism=1.

### 4.c HumanEval / GSM8K / BBH-mini fixtures cap at 5 tasks

The bench framework has full loaders for these but the on-disk
fixtures in `data/benchmarks/` only contain 5 tasks each. Scaling
`--tasks-per-benchmark` beyond 5 only affects `synthetic_routing`
(generated on the fly).

**Fix (low priority):** download the public datasets and trim to
~50-100 tasks each. See `flybrain/benchmarks/loaders/*.py`.

### 4.d DataSphere not applied

`infra/terraform/datasphere.tf` requires `organization_id` +
`billing_account_id` from the YC tenant. The skeleton is correct
(per PLAN.md §622) but has not been validated against a live run.
See `docs/yandex_setup.md` for how to find these IDs.

## 5. How to extend (common follow-ups)

### Add a new baseline

1. Add a `BaselineSpec` to `flybrain/baselines/registry.py` (factory +
   description + `mas_config_overrides` if needed).
2. Add it to one of the suites (`full_min`, `static`,
   `embedding_ablation`, `verifier_ablation`, `training_ablation`).
3. Write a unit test in `tests/python/test_baselines.py`.
4. `make test && make lint`.

### Add a new benchmark

1. Add a loader in `flybrain/benchmarks/loaders/` that returns
   `list[Task]`.
2. Wire it into `flybrain/benchmarks/runner.py::_load_benchmark`.
3. Add fixtures or generation logic.
4. Add eval-side scoring in `flybrain/eval/metrics.py` if it needs a
   custom verifier.

### Add a new training phase

1. New module under `flybrain/training/`.
2. Driver script under `scripts/run_<phase>.py` that takes a
   controller + dataset + checkpoint output path.
3. Wire into `_flybrain_with_checkpoint` factory in `registry.py`
   (use the env-var pattern from existing phases).
4. Update §4.a above as you fix the dim mismatch.

## 8. Conventions & gotchas

* **Style:** ruff + mypy + clippy. `make lint` runs all three.
  Pre-commit hooks are installed via `pre-commit install`.
* **Docstrings:** use Google-style for Python; standard for Rust.
  Avoid emojis in code/docs.
* **Branch naming:** `devin/<unix-ts>-<topic>` for Devin sessions;
  `<author>/<topic>` for humans.
* **Commit messages:** plain English, ≤72 char subject + body
  paragraph if non-trivial. The PR template lives in
  `.github/PULL_REQUEST_TEMPLATE.md`.
* **`@dataclass(slots=True)`** is used everywhere for hot dataclasses.
  This means **no `__dict__`** — use `dataclasses.asdict(cfg)` if you
  need to serialize. Both training scripts had this bug; fixed in
  PR #8.
* **Cache files:** `data/cache/` and per-bench
  `<output>/yandex_cache.sqlite` are gitignored. Cache hits are free
  and don't count against the LLM budget.
* **Costs:** always log `cost_rub` in trace steps. Never expose
  costs in cents/USD — the rest of the codebase assumes RUB.

### 11.f Rotation / revocation

* **`YANDEX_API_KEY`**: rotate every 90 days. Old key:
  `yc iam api-key delete <key-id>`. New key:
  `yc iam api-key create --service-account-id ajemvvngo4s4gjai7p2r`.
* **`YC_TOKEN`**: expires automatically every 12 h; nothing to do.
* **`github_token`**: GitHub auto-revokes inactive fine-grained PATs
  after 1 year; rotate before then via
  https://github.com/settings/tokens.
* **If a key leaks**: `yc iam api-key delete --id <leaked-id>` first,
  then `yc iam access-key delete` for the static S3 access key, then
  `yc iam access-key create` to generate a new one. Update Terraform
  state with `terraform apply` (the static key will rotate
  automatically because it's managed by the
  `yandex_iam_service_account_static_access_key` resource).
