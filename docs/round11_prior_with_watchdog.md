# Round 11 — Null-prior × watchdog v2 cross-bench (0 ₽)

**Status:** completed — N=10 × 4 benchmarks × 5 baselines = 200 task-runs.
**Budget:** 0 ₽ (free-tier OpenRouter, round-7+ hard constraint).
**PR:** updates [#14](https://github.com/blessblissmari/The-fly-brain-test/pull/14)
on branch `devin/1777721760-trained-baselines-prior-graph`.

## 1  Motivation

Round 10 ran a controlled null-model ablation on the connectome
prior used by `flybrain_sim_pretrain` and arrived at a *partial
falsification* of README §17:

> "FlyBrain prior helps **because it is biological**"

was not supported by the data — the trained controller is
insensitive to swapping the FlyWire 783 prior for ER, shuffled
(degree-preserving Maslov-Sneppen) or reverse (transpose) priors
(all four overall ≈ 0.775-0.800, Wilcoxon p > 0.7 after
Bonferroni).

That round-10 ablation was on the *raw* `flybrain_sim_pretrain`
checkpoint — i.e. **without** the round-7/8/9 finalizer-watchdog
wrapper. But by round-9 we knew the watchdog v2/v3 was the
single biggest quality lever in the whole project: it closed the
synthetic_routing gap from 0.500 to 1.000 and the humaneval gap
from 0.600 to 0.900 (`docs/round9_autotuned.md` §4.1).

So a natural follow-up question that the team raised
[in Telegram](https://t.me/) on 2026-05-06 is:

> "Maybe biology *does* help, but only once the watchdog scaffold
>  is in place. With the wrapper supplying the missing Finalizer
>  step, does the controller now exploit FlyWire-specific topology
>  over null priors?"

Round 11 closes this gap with a single 200-task-run cross-bench
that wires every round-10 null-prior with the same round-8/9
watchdog v2 (per-task-type ``force_after`` dict).

## 2  Pre-registered hypotheses

Before running the live bench:

* **H1 (biology matters even with watchdog):**
  `real_fly+wd2 > er+wd2 ≈ shuffled+wd2 ≈ reverse+wd2`
  — biology adds value beyond what the scaffold provides; the
  prior is necessary for task-specific routing patterns the
  watchdog cannot supply.

* **H0 (scaffold is the value, prior is dispensable):**
  `real_fly+wd2 ≈ er+wd2 ≈ shuffled+wd2 ≈ reverse+wd2`
  — the watchdog is the universal rescuer; biology was a detour
  on the way to the right scaffolding.

* **H-direction (direction still matters with scaffold):**
  `real_fly+wd2 > reverse_fly+wd2`
  — biological direction is still informative even after
  watchdog termination is supplied.

These are mutually exclusive at the bench-level we can measure
with N=10 free-tier OpenRouter runs (95 % bootstrap CI half-width
≈ ±20 pp).

## 3  Methods

### 3.1 Baselines

| Baseline | Prior | Watchdog wrapper |
|---|---|---|
| `manual_graph` | n/a | n/a (control, hand-coded) |
| `flybrain_sim_pretrain_watchdog_v2` | FlyWire 783 K=64 Louvain | v2 (round-8) |
| `er_prior_watchdog_v2` | Erdős-Rényi K=64 (matched n_edges) | v2 |
| `shuffled_fly_watchdog_v2` | Maslov-Sneppen degree-preserving | v2 |
| `reverse_fly_watchdog_v2` | Adjacency transpose | v2 |

The watchdog v2 dict is held constant across all four
FlyBrain-architecture rows:

```python
force_after = {"coding": 28, "math": 12, "research": 16, "tool_use": 12}
stall_after = {"coding": 6,  "math": 3,  "research": 4,  "tool_use": 3}
```

These are the round-8 hand-tuned values, validated by round-9
auto-calibration to within ±2 of P90 of `manual_graph` traces.

### 3.2 Implementation

`flybrain/baselines/registry.py::_flybrain_with_checkpoint_and_watchdog`
gained a new `fly_graph_path: Path | None = None` keyword that
substitutes the canonical FlyWire prior at controller-construction
time without touching the watchdog wrapper. The substitution
flows through to both:

* the initial `AgentGraph` (via `flybrain.baselines.graphs.flybrain_prior_graph`),
* the cached `fly_vec` the controller reads each step (via
  `ControllerStateBuilder.fly_graph`).

The four round-11 baselines pass distinct `fly_graph_path` values
pointing at `data/flybrain/null_priors/{er_K64_seed0,
shuffled_K64_seed0, reverse_K64}.fbg` (round-10 artefacts).

### 3.3 Bench command

```bash
export OPENROUTER_API_KEY=...   # free-tier key OK
flybrain-py bench --suite round11_priors_with_watchdog \
  --backend openrouter --tasks-per-benchmark 10 \
  --seed 42 \
  --output data/experiments/bench_round11_priors_watchdog
```

### 4.3 Verdict (pre-registered selection rule)

**Partial H1 supported (biology partially matters even with watchdog):**

* real-fly+wd2 *strongly* beats truly random ER+wd2: **+17.5 pp**
  on overall (p = 0.044 uncorrected; 0.132 with Bonferroni — at
  N=10 we can detect a 17.5 pp effect but not call it significant
  after multiple comparisons).
* real-fly+wd2 ≈ shuffled+wd2 (Maslov-Sneppen
  degree-preserving; +2.5 pp).
* real-fly+wd2 ≈ reverse+wd2 (transpose / undirected adjacency;
  +2.5 pp, **direction immaterial under watchdog scaffold**).

**Where the gap lives.** Per-benchmark, real-fly+wd2 beats
er+wd2 almost entirely on humaneval (+80 pp: 1.000 vs 0.200).
On gsm8k, bbh_mini and synthetic_routing all four FlyBrain rows
are within 10 pp of each other.

**What this means for the README §17 claim.** Round-10 found
the *raw* GNN was insensitive to its prior (no significant
differences between any of er / shuffled / reverse / fly).
Round-11 shows that **with the watchdog scaffold in place**,
the biological prior *does* add value on coding tasks
specifically — it's not equally rescuable as ER. But the
degree distribution captures most of that gain
(real_fly ≈ shuffled), so the strong README claim "biological
because biological" is still oversold; the more defensible claim
is "biological topology has a degree distribution that helps
deep coding-style routing once a termination scaffold is
provided."

**Why humaneval specifically.** Coding tasks are the deepest
trajectories in the benchmark suite (manual_graph routes are
plan→code→test→debug→verify, ~20 LLM calls). With the watchdog
allowing 28 calls before forced termination, the controller has
*time* to leverage rich routing patterns that biological
topology encodes. Shorter benchmarks (bbh_mini ~3-4 calls,
gsm8k ~5 calls) finish before the prior-difference would
matter — they hit terminate naturally, not via watchdog
intervention.

## 6  Threats to validity

* **Free-tier model rotation** — same caveat as round-10 §6:
  cross-baseline comparisons within one bench dir share the same
  upstream rotation and are valid; cross-round absolute numbers
  are not directly comparable to round-9 / round-10.
* **N=10 is small** — 95 % bootstrap CI half-widths on a binary
  success rate at N=10 are ±~20 pp. If round-11 shows H0 it
  could mask a real but small biological effect that round-13's
  paid YandexGPT bench (N=10 with reproducible upstream) might
  uncover.
* **Same-checkpoint-different-prior caveat (inherited from round-10)** —
  this is an *inference-time* ablation. The checkpoint
  `sim_pretrain_gnn_v6.pt` was trained with the canonical FlyWire
  prior and `fly_dim=8` zeroed (`flybrain.training.simulation_pretrain`),
  so the result speaks only to what the trained weights do when
  the prior is swapped at evaluation. A round-12+ retraining
  ablation (training from scratch on each null prior) would test
  the stronger claim that the controller cannot exploit biology
  even when given the chance.
